01

Define the wire once

Contracts

A contract declares the HTTP method, path, boundary types, response, status, and possible application errors. Any type Pydantic can validate can sit at the boundary. 

from pydantic import BaseModel, Field
from tenchi.contracts import contract

class CreateTodo(BaseModel):
    title: str = Field(min_length=1)

class Todo(BaseModel):
    id: str
    title: str
    completed: bool

create_todo_contract = contract(
    method="POST",
    path="/todos",
    request=CreateTodo,
    response=Todo,
    status=201,
    summary="Create a todo",
    tags=("todos",),
)
Composition-time checks. Route parameters and the return annotation must exactly match the contract, so broken wiring fails before the server starts.
02

Keep the center boring

Use cases & ports

Use cases are plain async functions. Ports are protocols. A frozen context carries the implementations chosen by the application. 

from typing import Protocol

class TodoRepository(Protocol):
    async def create(
        self, *, title: str
    ) -> Todo: ...
from dataclasses import dataclass

@dataclass(frozen=True, slots=True)
class AppContext:
    todos: TodoRepository
async def create_todo(
    request: CreateTodo,
    context: AppContext,
) -> Todo:
    return await context.todos.create(title=request.title)

Use-case tests can pass a memory adapter directly. No HTTP client, framework base class, or dependency container is required.

03

Bind, compose, run

Application wiring

Routes bind contracts to use cases. Route groups compose features, shared errors, the OpenAPI document, and health checks. The server receives concrete wiring at the composition root. 

from tenchi.health import health_route
from tenchi.openapi import openapi_route
from tenchi.routes import route, route_group
from tenchi.server import create_app

todo_routes = route_group(
    route(create_todo_contract, create_todo),
)

routes = route_group(
    todo_routes,
    openapi_route(todo_routes, title="Todos", version="1.0.0"),
    health_route(),
)

app = create_app(
    routes=routes,
    context_factory=lambda: AppContext(todos=repository),
)

Lifespan resources

Pass an async context manager as lifespan; its value is supplied to the context factory.

Request scope

Make the context factory an async context manager for commit-on-success and rollback-on-error semantics.

Boundary hooks

Hooks receive RequestInfo and context before the use case—ideal for authentication.

Middleware

Pass ordinary Starlette middleware without wrapping or hiding the ASGI runtime.

04

Errors are part of the contract

Errors & auth

Define stable application errors once and declare them on the contract or route group. An undeclared AppError becomes a framework-owned 500 rather than leaking an undocumented response. 

from tenchi.errors import AppError, ErrorDef

todo_not_found = ErrorDef(
    code="TODO_NOT_FOUND",
    status=404,
    message="Todo not found",
)

get_todo_contract = contract(
    method="GET",
    path="/todos/{todo_id}",
    params=GetTodoParams,
    response=Todo,
    errors=(todo_not_found,),
)

async def get_todo(params: GetTodoParams, context: AppContext) -> Todo:
    todo = await context.todos.get(params.todo_id)
    if todo is None:
        raise AppError(todo_not_found, details={"todo_id": params.todo_id})
    return todo
Authentication in hooks; authorization in use cases. Hooks attach identity to a replaced context. Pure policy functions decide what that identity may do with a domain subject.
05

The contract travels

Typed client & OpenAPI

The async client serializes inputs and validates responses from the same contract. Declared error envelopes return as AppError; everything else is an UnexpectedResponseError. 

from tenchi.client import Client

async with Client(base_url="https://api.example.com") as client:
    todo = await client.call(
        create_todo_contract,
        request=CreateTodo(title="Buy milk"),
    )

assert isinstance(todo, Todo)

openapi_route(...) serves an OpenAPI 3.1 document from the same declarations. Schema names, aliases, path converters, media types, errors, deprecation, and sunset metadata stay aligned with the server and client.

Use the same boundary off HTTP

Workers & pagination

Execute a use case anywhere

execute() validates raw job data against the use case’s request annotation and applies the same context scoping used by the server. 

from tenchi.execution import execute

await execute(
    notify_member_added,
    request_json=job.payload,
    context=context,
)

Return typed pages

Subclass PageQuery for filters, declare Page[Item], and preserve the requested window with page(). 

class ListTodosQuery(PageQuery):
    completed: bool | None = None

return page(
    items,
    total=total,
    query=query,
)
07

No server process required

Testing

open_client provides the typed client in-process; open_http exposes raw httpx responses. Both run the app lifespan around the test. 

from tenchi.testing import open_client, open_http

async with open_client(app) as client:
    todo = await client.call(
        create_todo_contract,
        request=CreateTodo(title="Buy milk"),
    )

async with open_http(app) as http:
    response = await http.get("/health")
    assert response.status_code == 200
